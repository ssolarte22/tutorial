Almacenamiento Local (Web Storage)

1) ¿Qué es el Web Storage API?

HTML5 introdujo una forma mucho más limpia y potente de guardar datos en el navegador del usuario en comparación con las antiguas cookies. Permite almacenar información en formato clave-valor y tiene una capacidad mucho mayor (normalmente 5MB).

Se divide en dos tipos principales:

  • localStorage: Los datos persisten incluso si cierras el navegador o apagas el ordenador. No tienen fecha de caducidad.
  • sessionStorage: Los datos se borran en cuanto se cierra la pestaña o ventana del navegador.

2) Métodos principales (JavaScript)

Para interactuar con el Storage debes usar JavaScript. Los métodos son idénticos para local y session: 

// Guardar un dato
localStorage.setItem('usuario', 'Juan Pérez');

// Leer un dato
let nombre = localStorage.getItem('usuario');

// Borrar un dato específico
localStorage.removeItem('usuario');

// Borrar TODOS los datos
localStorage.clear();

4) Uso común de Storage

  • Modo Oscuro/Claro: Guardar la preferencia del tema del usuario.
  • Sesiones de usuario: Guardar tokens de autenticación (JWT).
  • Carrito de compras: Mantener productos seleccionados si el usuario abandona la página accidentalmente.
  • Importante: Nunca guardes contraseñas ni datos altamente sensibles aquí, ya que pueden ser leídos por scripts del lado del cliente.
