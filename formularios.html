Formularios en HTML5

1) Nuevos tipos de Input y Autovalidacion

HTML5 introdujo muchos tipos nuevos para la etiqueta <input> (como email, date, color, range) y atributos especiales de validacion (como required, pattern, min, max).

Esto permite que el navegador valide los datos introducidos por el usuario antes de enviar el formulario, reduciendo la necesidad de escribir validaciones complejas con JavaScript.

2) Demo: Formulario de Registro basico

Intenta enviar el formulario dejandolo vacio, o escribiendo un texto que no sea un email en el campo correspondiente. El propio navegador mostrara tooltips de error gracias al atributo required y a type="email".

3) Demo: Inputs visuales (Color, Rango y Fecha)

Dependiendo del navegador (y soporte del sistema operativo), estos inputs muestran interfaces avanzadas (como calendarios nativos o paletas de colores) listas para usar.

4) Pseudo-clases CSS para validacion

Puedes dar formato visual inmediato a los inputs dependiendo de si sus datos son validos o no, utilizando las pseudo-clases :valid y :invalid en CSS. Si revisas el CSS de este ejemplo, veras que los campos obligatorios incompletos o incorrectos tienen un borde rojo.
